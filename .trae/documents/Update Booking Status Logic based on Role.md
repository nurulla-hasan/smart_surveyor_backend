# বুকিং স্ট্যাটাস লজিক আপডেট প্ল্যান

আপনার রিকোয়েস্ট অনুযায়ী, অ্যাডমিন বা সার্ভেয়ার বুকিং দিলে সেটি সরাসরি **`scheduled`** হবে এবং ক্লায়েন্ট বুকিং দিলে সেটি **`pending`** হবে। আপনার কোডবেসে এই লজিকটি অলরেডি ইমপ্লিমেন্ট করা আছে, তবে আমি এটি আরও একবার নিশ্চিত করে দিচ্ছি।

## বর্তমান লজিক ভেরিফিকেশন:
[booking.controller.ts](file:///c:/Users/golap/Desktop/smart-surveyor-api/src/controllers/booking.controller.ts) এর `createBooking` ফাংশনে নিচের লজিকটি কাজ করছে:

1. **Client Role চেক:** যদি রিকোয়েস্ট পাঠানো ইউজারের রোল `client` হয়:
   - স্ট্যাটাস সব সময় **`pending`** সেট করা হয়।
   - বুকিংটি নির্দিষ্ট `surveyorId` এর আন্ডারে অ্যাসাইন হয়।

2. **Admin/Surveyor Role চেক:** যদি ইউজারের রোল `admin` অথবা `surveyor` হয়:
   - স্ট্যাটাস ডিফল্টভাবে **`scheduled`** সেট করা হয় (যদি রিকোয়েস্ট বডিতে অন্য কোনো স্ট্যাটাস না থাকে)।

## টেকনিক্যাল ইমপ্লিমেন্টেশন (যা অলরেডি কোডে আছে):
```typescript
if (role === 'client') {
  finalStatus = 'pending'; // ক্লায়েন্টের জন্য সব সময় pending
} else {
  // অ্যাডমিন বা সার্ভেয়ারের জন্য scheduled ডিফল্ট
  finalStatus = (status as BookingStatus) || 'scheduled'; 
}
```
