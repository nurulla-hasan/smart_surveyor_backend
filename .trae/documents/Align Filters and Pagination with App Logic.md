# Backend Pagination এবং Filter Logic আপডেট করার পরিকল্পনা

আপনার অ্যাপ্লিকেশনের লজিক এবং স্ক্রিনশটের সাথে সামঞ্জস্য রেখে আমি নিম্নলিখিত পরিবর্তনগুলো করার পরিকল্পনা করছি:

## ১. Pagination Response Structure আপডেট
বর্তমানে পেজিনেশন ডাটা সরাসরি `data` অবজেক্টের ভেতর থাকে। আপনার অনুরোধ অনুযায়ী আমি এটিকে একটি `meta` অবজেক্টের ভেতর নিয়ে আসব এবং আরও বর্ণনামূলক (descriptive) নাম ব্যবহার করব।

- **নতুন ফরম্যাট:**
  ```json
  "data": {
    "bookings": [...],
    "meta": {
      "totalItems": 100,
      "totalPages": 10,
      "currentPage": 1,
      "pageSize": 10
    }
  }
  ```
- এই পরিবর্তনটি আমি ৩টি কন্ট্রোলারে করব:
  1. `booking.controller.ts`
  2. `client.controller.ts`
  3. `report.controller.ts`

## ২. Booking Filter Logic যাচাই ও আপডেট
আপনার স্ক্রিনশটে "Upcoming", "Requests", "History" ট্যাব দেখা যাচ্ছে। আমাদের ব্যাকএন্ডের লজিক বর্তমানে এভাবেই কাজ করছে:
- `upcoming` -> Upcoming ট্যাব (ভবিষ্যতের শিডিউল করা বুকিং)
- `pending` -> Requests ট্যাব (যেগুলো এখনো অ্যাপ্রুভ করা হয়নি)
- `past` -> History ট্যাব (অতীতের সম্পন্ন করা বুকিং)

আমি পোস্টম্যান কালেকশনে এই ফিল্টারগুলোর বর্ণনা আরও পরিষ্কার করে দেব যাতে আপনার অ্যাপের লজিকের সাথে হুবহু মিলে যায়।

## ৩. Postman Collection আপডেট
- `Get All Bookings` এবং অন্যান্য পেজিনেটেড রিকোয়েস্টের ডিসক্রিপশন এবং কুয়েরি প্যারামিটার আপডেট করা হবে।
- রেসপন্স এক্সাম্পলগুলো নতুন `meta` স্ট্রাকচার অনুযায়ী আপডেট করা হবে।

আপনি অনুমতি দিলে আমি কোড এডিট করা শুরু করব।
